HTML is the bare minimum technology needed to get a website up and running ... establishes the structure and layout of a given website as well as providing the bulk of its content.

CSS (used for styling a webpage) and JavaScript (makes web pages interactable) are optional compared to HTML.

A fully functioning website doesn't need JavaScript to work. However, JavaScript is widely considered the most important piece or web development technology out there ... so much so that your capacity to create, understand, and explain JavaScript will likely be one of the key reasons you get hired in the future.

JavaScript (along with HTML and CSS) is the only programming language that can run in the browser ... other pseudo-languages that run in the browser (such as PHP) are ultimately converted into either HTML, CSS, or JS somewhere down the line.

Here's the typical syntax for creating a JavaScript variable:

        
            var character = "Snow White" ;

In the above code snippet ...

var // stands for Var Keyword // Tells the browser that you want to create a JavaScript variable.

character // stands for Variable Name // Assigns a name to your variable. In this example the variable name is character, but you can come up with any name you want!

= // stands for Assignment // Assigns the variable a specific value.

"Snow White" // stands for Value // The specific quality or distinction that something takes on.

; // stands for Termination // Marks the end of the statement.

*NOTE_01: As of 2015, JavaScript no longer requires the ; syntax for a code block to function ... however, including it makes JS code easier to read, so continuing to use it might still be worth it.

*NOTE_02: The above note also applies to the deprication of var, we'll get into alternatives for using that syntax later in this webpage.

Two methods you can use to add JavaScript code to your HTML document include doing it "In-Line" which means that you wrap the JS code between two HTML script tags, and "Externally" by writing a script tag with a src="" attribute. For example:

        
            +script src="script.js"+ +/script+

*NOTE_03: Please substitute any "+" you see within certain code snippets with the appropriate angle brackets that are typically needed for HTML code to function (see the example above). Unfortunately, the code won't display correctly if I include angle brackets, so "+" is being used in the meantime while solutions are explored.
